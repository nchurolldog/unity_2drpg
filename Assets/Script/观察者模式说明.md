# 观察者模式说明 —— ExpManager 与 Enemy_Health

## 涉及脚本

| 脚本 | 路径 | 角色 |
|------|------|------|
| **Enemy_Health** | `Script/Enemy/Enemy_Health.cs` | 事件发布者（Subject） |
| **ExpManager** | `Script/Player/ExpManager.cs` | 事件订阅者（Observer） |
| **MeleeAttack** | `Script/Combat/MeleeAttack.cs` | 触发者（调用方，不参与模式本身） |

## 运行流程图

```
玩家攻击
  → MeleeAttack.DealDamage()
    → Enemy_Health.ChangeHealth(-Damage)     // 扣血
      → 血量 ≤ 0
        → OnMonsterDefeated?.Invoke(expReward)  // 广播"怪物死了"事件
          → ExpManager.GainExp(exp)              // 收到事件，加经验
          → Destroy(gameObject)                  // 销毁怪物
```

## 核心概念：委托与事件

### 1. 委托（delegate）—— 定义"方法签名"

```csharp
// Enemy_Health.cs 第7行
public delegate void MonsterDefeated(int exp);
```

**通俗理解：** 委托就是一个"方法模板"。它规定：任何想响应"怪物被击败"这件事的方法，**必须**满足 `void 方法名(int)` 这个签名——无返回值，接收一个 int 参数。

### 2. 事件（event）—— 广播站

```csharp
// Enemy_Health.cs 第8行
public static event MonsterDefeated OnMonsterDefeated;
```

**通俗理解：** 事件是一个"广播频道"。`static` 表示这个频道是全局的，所有敌人共用同一个频道。任何人想听广播就"插上线"（+=订阅），不想听了就"拔掉"（-=取消订阅）。

### 3. 发布（Invoke）—— 喊话

```csharp
// Enemy_Health.cs 第23行
OnMonsterDefeated?.Invoke(expReward);
```

**通俗理解：** 怪物死的时候对着频道喊一声"我死了！经验值拿去！"  
`?.Invoke()` 中的 `?.` 是**空条件运算符**——如果频道没人收听（事件为 null），就不喊话，避免报错。

### 4. 订阅（+=）与取消订阅（-=）—— 收听

```csharp
// ExpManager.cs 第19行
private void OnEnable()
{
    Enemy_Health.OnMonsterDefeated += GainExp;   // 开始收听
}

// ExpManager.cs 第24行
private void OnDisable()
{
    Enemy_Health.OnMonsterDefeated -= GainExp;   // 停止收听
}
```

**为什么要取消订阅？** ExpManager 挂载在玩家身上。如果玩家死亡/销毁时不取消订阅，事件频道里还保留着对已销毁对象的引用，会导致：
- 内存泄漏（垃圾回收无法释放 ExpManager）
- 空引用异常（怪物死亡时试图调用已销毁对象的方法）

**为什么用 OnEnable/OnDisable 而不是 Start/OnDestroy？** 这是更稳健的做法。物体被禁用（SetActive(false)）时也会取消订阅，重新启用时再次订阅，避免被禁用的物体收到事件。

## 观察者模式的好处

```
❌ 不用观察者模式（紧耦合）：
Enemy_Health 需要直接持有 ExpManager 引用，每死一个怪物就调用 expManager.GainExp()。
→ 怪物类必须"认识"经验系统，改一个就要改另一个。

✅ 用观察者模式（松耦合）：
Enemy_Health 只管喊话，不管谁在听。
ExpManager 只管收听，不管谁在喊。
→ 两者互不依赖，各自独立开发、独立测试。
```

## 如何新增一个"听众"

比如你想让怪物死亡时也播放音效，只需在 `SoundManager` 里写：

```csharp
private void OnEnable()
{
    Enemy_Health.OnMonsterDefeated += PlayDeathSound;
}

private void OnDisable()
{
    Enemy_Health.OnMonsterDefeated -= PlayDeathSound;
}

private void PlayDeathSound(int exp)
{
    // 播放音效，exp 参数用不到就直接忽略
    audioSource.PlayOneShot(deathClip);
}
```

**Enemy_Health 不需要任何修改。** 这就是观察者模式的核心价值。

## 关键注意事项

1. **事件是 `static` 的** —— 所有敌人实例共享同一个事件。这意味着 ExpManager 只需订阅一次，就能收到所有敌人的死亡通知。
2. **`GainExp(int)` 签名匹配 `MonsterDefeated(int)`** —— 编译器检查参数类型和返回值必须一致。
3. **在 `OnDisable` 中取消订阅** —— 这是最容易忘记的坑，忘了就会产生"幽灵调用"。
